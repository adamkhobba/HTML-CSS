Thread

Defintion

Threads in C are a way to achieve concurrent execution of code, allowing a program to perform multiple operations simultaneously. This is particularly useful for tasks that can be executed independently, such as handling multiple client connections in a server or performing background computations while maintaining responsiveness in the main application. created byedsger Dijkstra

Parallelism vs concurrency

Concurrent and parallel are effectively the same principle as you correctly surmise, both are related to tasks being executed simultaneously although I would say that parallel tasks should be truly multitasking, executed "at the same time" whereas concurrent could mean that the tasks are sharing the execution thread while still appearing to be executing in parallel.

Multi-threading Models

Many-to-One Model

  • Many user-level threads mapped to single kernel thread
  • One thread blocking causes all to block
  • Thread management done by user-level threads library

One-to-One Model

  • Each user-level thread maps to kernel thread
  • One thread blocking does not cause others to block
  • Thread management done by kernel

Many-to-Many Model

  • Many user-level threads mapped to many kernel threads
  • Allows the operating system to create a sufficient number of kernel threads
  • Thread management done by user-level threads library

Mutual exclusion (Mutex)

Definition

A mutex is a lock that is used to protect shared data from being simultaneously accessed by multiple threads. When a thread acquires a mutex, it has exclusive access to the shared data until it releases the mutex. This prevents other threads from accessing the shared data while it is being modified, ensuring that the data remains consistent.

Dead lock

Definition

A deadlock is a situation in which two or more threads are waiting for each other to release a resource that they need, preventing all of them from making progress. Deadlocks can occur when multiple threads acquire locks in a different order, creating a circular dependency that prevents any of them from releasing their locks.

Semaphore

Definition

A semaphore is a synchronization primitive that is used to control access to a shared resource by multiple threads. It maintains a count that represents the number of available resources, and provides operations to acquire and release resources. When a thread acquires a semaphore, it decrements the count, and when it releases the semaphore, it increments the count.

named semaphore

Definition

A named semaphore is a semaphore that is identified by a unique name in the system, allowing multiple processes to access the same semaphore. This is useful for coordinating access to shared resources between different processes, such as in a client-server application.

unnamed semaphore

Definition

An unnamed semaphore is a semaphore that is not identified by a unique name in the system, and is used for synchronization between threads within the same process. This is useful for coordinating access to shared resources between different threads in a multi-threaded application.
